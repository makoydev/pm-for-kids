# Feature Checklist

This checklist tracks the next features for PM for Kids. The goal is to keep each addition tied to a learning outcome, not just add controls.

## Added

- [x] Adaptive coach panel that suggests the next useful project action.
- [x] Immediate assignment feedback for capacity, skill fit, and start readiness.
- [x] Scenario choice effect chips for visible tradeoffs.
- [x] Optional activity log for reviewing decisions.

## Next

- [ ] Starter checklist for first-run guidance.
  - Acceptance: players can see a short list of early actions and which ones they have completed.
  - Learning goal: teach the project loop of assign, start, manage risk, review, and advance time.
- [ ] Glossary panel for PM terms.
  - Acceptance: players can open plain-language definitions for scope, risk, dependency, stakeholder, quality, budget, and morale.
  - Learning goal: connect game actions to project management vocabulary.
- [ ] Difficulty levels.
  - Acceptance: easy mode shows full effect previews; challenge mode hides exact numbers and uses softer hints.
  - Learning goal: support younger learners while leaving room for replayability.
- [ ] Keyboard support pass.
  - Acceptance: primary controls are reachable and usable by keyboard, including dialogs and task actions.
  - Learning goal: improve accessibility and classroom usability.

## Later

- [ ] More project scenarios, starting with a birthday party, class garden, robot demo, or bake sale.
- [ ] Scenario data extraction into JSON so new projects can be authored without editing app logic.
- [ ] Parent or teacher summary with discussion prompts after the retrospective.
- [ ] Replay settings for event randomness and optional risk frequency.
