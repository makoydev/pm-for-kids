# UX Review Notes

## May 2026 Pass

This pass focused on keeping the playable prototype understandable for kids who are learning project management concepts for the first time.

## Improvements Made

- Added an adaptive coach panel above the board with the next useful action, current task progress, remaining coins, and week status.
- Fixed assignee changes so the task card updates immediately after choosing a teammate. This makes the Start button, skill fit hints, and capacity warnings appear without a page refresh.
- Added effect chips to scenario choices so players can quickly compare tradeoffs like quality, morale, trust, and coins.
- Kept the activity log optional so the main board stays focused while still preserving a history for review.

## UX Principles

- Make the next action obvious without removing player choice.
- Show tradeoffs before the player commits to a PM decision.
- Use plain language and visible cause-and-effect feedback.
- Keep advanced PM ideas, such as risk mitigation and dependencies, attached to concrete project actions.

## Follow-Up Opportunities

- Add scenario difficulty levels so younger players can keep full effect previews while older players get less certainty.
- Add keyboard shortcuts for moving through the board once the core controls stabilize.
- Add a glossary panel for terms like scope, dependency, risk, stakeholder, and quality.
- Add a short first-run checklist for classrooms or parent-guided play.
